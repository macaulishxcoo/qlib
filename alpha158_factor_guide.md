### 手工验算示例

以 KMID（因子1）为例：
```
KMID = ($close - $open) / $open
     = (7.703750 - 7.782040) / 7.782040
     = -0.078290 / 7.782040
     = -0.010060  ✓
```

以 ROC5（因子14）为例（5 日前的收盘价 / 今日收盘价）：

5 日前是 2020-09-14，收盘 = 7.750724。注意 09-19 和 09-20 是周末，所以往前数 5 个交易日：
```
ROC5 = Ref($close, 5) / $close
     = 7.750724 / 7.703750
     = 1.006098  ✓
```

你可以用 Excel 或 Python 验证其余因子。

## 一、K线形态因子（9个）

基于当日 OHLC 四个价格，描述单根 K 线的形态特征。

| 序号 | 因子名 | 公式 | 含义 | 示例值 |
|:--:|--------|------|------|:--:|
| 1 | **KMID** | `($close - $open) / $open` | 实体涨跌幅。正值=阳线，负值=阴线，绝对值越大实体越长 | -0.010060 |
| 2 | **KLEN** | `($high - $low) / $open` | 日内振幅。全日最高最低之间的波动幅度 | 0.014085 |
| 3 | **KMID2** | `($close - $open) / ($high - $low)` | 实体占全日振幅的比例。接近0=十字星，接近±1=光头光脚 | -0.714281 |
| 4 | **KUP** | `($high - max($open, $close)) / $open` | 上影线长度。最高价距离实体上沿的距离 | 0.003018 |
| 5 | **KUP2** | `($high - max($open, $close)) / ($high - $low)` | 上影线占比。上影线占全日振幅的比例 | 0.214289 |
| 6 | **KLOW** | `(min($open, $close) - $low) / $open` | 下影线长度。实体下沿距离最低价的距离 | 0.001006 |
| 7 | **KLOW2** | `(min($open, $close) - $low) / ($high - $low)` | 下影线占比。下影线占全日振幅的比例 | 0.071430 |
| 8 | **KSFT** | `(2 * $close - $high - $low) / $open` | 收盘价在最高最低之间的位置。0=居中，正=偏上 | -0.012072 |
| 9 | **KSFT2** | `(2 * $close - $high - $low) / ($high - $low)` | 收盘位置占比。1=收在最高，-1=收在最低，0=居中 | -0.857140 |


### 信号解读

#### 1. KMID — 实体涨跌幅

| 值范围 | K线形态 | 市场信号 |
|--------|---------|----------|
| > +0.03 | 大阳线（涨幅 3%+） | 强烈多头信号，买方力量远超卖方 |
| +0.01 ~ +0.03 | 中阳线 | 多方占优，上涨趋势温和延续 |
| 0 ~ +0.01 | 小阳线 | 多空接近平衡，多方略占优 |
| -0.01 ~ 0 | 小阴线 | 多空接近平衡，空方略占优 |
| -0.03 ~ -0.01 | 中阴线 | 空方占优，下跌趋势温和延续 |
| < -0.03 | 大阴线（跌幅 3%+） | 强烈空头信号，卖方力量远超买方 |

> **注意**：绝对值越大，趋势越强；绝对值极小（≈0）时为十字星，可能变盘。

#### 2. KLEN — 日内振幅

| 值范围 | 形态特征 | 市场信号 |
|--------|---------|----------|
| > 0.05 | 超长振幅（5%+） | 剧烈波动，通常伴随重大消息或情绪极端变化 |
| 0.03 ~ 0.05 | 较大振幅 | 多空分歧明显，活跃交易日 |
| 0.01 ~ 0.03 | 正常振幅 | 常规波动，市场情绪平稳 |
| 0 ~ 0.01 | 极小振幅（窄幅） | 多空僵持、观望情绪浓厚，往往是变盘前兆 |

> **注意**：KLEN 只衡量波动幅度，不区分涨跌方向。高 KLEN + 小 KMID = 长上下影线十字星（多空博弈剧烈）。

#### 3. KMID2 — 实体占振幅比例

| 值范围 | K线形态 | 市场信号 |
|--------|---------|----------|
| 接近 +1 | 光头光脚大阳线 | 多方完全控盘，最强看涨信号 |
| +0.5 ~ +1 | 长实体阳线（影线较短） | 多方强势，影线越短信号越纯 |
| 0 ~ +0.5 | 短实体阳线（影线较长） | 多方占优但受阻力，上影越长抛压越重 |
| ≈ 0 | 十字星 | 多空均衡，方向不明，变盘信号 |
| -0.5 ~ 0 | 短实体阴线（影线较长） | 空方占优但受支撑，下影越长承接越强 |
| -1 ~ -0.5 | 长实体阴线（影线较短） | 空方强势，影线越短信号越纯 |
| 接近 -1 | 光头光脚大阴线 | 空方完全控盘，最强看跌信号 |

> **注意**：与 KMID 配合使用。KMID 看绝对大小（涨跌幅多少），KMID2 看相对大小（实体在日内占比多少）。

#### 4. KUP — 上影线长度（相对开盘价）

| 值范围 | K线形态 | 市场信号 |
|--------|---------|----------|
| = 0 | 光头（无上影线） | 盘中最高价 = 实体上沿，多方未遇上方抛压 |
| 0 ~ 0.005 | 短上影线 | 上方抛压较轻，多方占主导 |
| 0.005 ~ 0.015 | 中等上影线 | 上方有一定抛压，多头冲高后回落 |
| > 0.015 | 长上影线 | 上方抛压沉重，多方进攻被打回，看跌信号 |

#### 5. KUP2 — 上影线占振幅比例

| 值范围 | K线形态 | 市场信号 |
|--------|---------|----------|
| = 0 | 光头 | 无上影线，多方完全控盘至最高 |
| 0 ~ 0.2 | 短上影（占比 <20%） | 上影线不重要，主要看实体方向 |
| 0.2 ~ 0.5 | 中等上影（占比 20%~50%） | 多方冲高后明显回落 |
| 0.5 ~ 0.8 | 长上影（占比 50%~80%） | 上影线主导 K 线形态，冲高回落严重 |
| > 0.8 | 射击之星 / 倒锤头 | 极度看跌信号（阳线形态）或底部反转试探（阴线形态需配合位置判断） |

> **注意**：KUP2 高值 + KMID 正值 = 上吊线（高位出现看跌）；KUP2 高值 + KMID 负值 = 倒锤头（低位出现可能反转）。

#### 6. KLOW — 下影线长度（相对开盘价）

| 值范围 | K线形态 | 市场信号 |
|--------|---------|----------|
| = 0 | 光脚（无下影线） | 盘中最低价 = 实体下沿，空方未遇下方承接 |
| 0 ~ 0.005 | 短下影线 | 下方承接较轻，空方占主导 |
| 0.005 ~ 0.015 | 中等下影线 | 下方有一定承接，空头下探后反弹 |
| > 0.015 | 长下影线 | 下方承接强劲，空方进攻被拉回，看涨信号 |

#### 7. KLOW2 — 下影线占振幅比例

| 值范围 | K线形态 | 市场信号 |
|--------|---------|----------|
| = 0 | 光脚 | 无下影线，空方完全控盘至最低 |
| 0 ~ 0.2 | 短下影（占比 <20%） | 下影线不重要，主要看实体方向 |
| 0.2 ~ 0.5 | 中等下影（占比 20%~50%） | 空方下探后明显反弹 |
| 0.5 ~ 0.8 | 长下影（占比 50%~80%） | 下影线主导 K 线形态，下探回升明显 |
| > 0.8 | 锤子线 / 上吊线 | 极度看涨信号（阴线形态）或顶部反转试探（阳线形态需配合位置判断） |

> **注意**：KLOW2 高值 + KMID 正值 = 锤子线（低位出现看涨）；KLOW2 高值 + KMID 负值 = 上吊线（高位出现看跌）。

#### 8. KSFT — 收盘价在日内区间的位置（相对开盘价归一化）

| 值范围 | K线形态 | 市场信号 |
|--------|---------|----------|
| > +0.02 | 收盘明显偏上 | 多方强势，趋势延续信号（次日大概率继续上涨） |
| +0.005 ~ +0.02 | 收盘略偏上 | 多方略占优 |
| -0.005 ~ +0.005 | 收盘居中 | 多空均衡，方向不明 |
| -0.02 ~ -0.005 | 收盘略偏下 | 空方略占优 |
| < -0.02 | 收盘明显偏下 | 空方强势，趋势延续信号（次日大概率继续下跌） |

> **注意**：KSFT 分母是 `$open`，受绝对股价影响。10 元股票和 100 元股票同样的收盘位置，KSFT 值差 10 倍。跨股票比较时用 KSFT2。

#### 9. KSFT2 — 收盘位置占比（标准化到 [-1, 1]）

| 值范围 | K线形态 | 市场信号 |
|--------|---------|----------|
| +0.8 ~ +1.0 | 光头阳线（收盘接近最高） | 多方完全控盘，最强看涨信号 |
| +0.5 ~ +0.8 | 高位收盘阳线 | 多方强势，可能有短上影线 |
| +0.2 ~ +0.5 | 阳线但收盘偏中上 | 多方占优，但尾盘有回落 |
| -0.2 ~ +0.2 | 十字星 / 收盘居中 | 多空均衡，变盘信号 |
| -0.5 ~ -0.2 | 阴线但收盘偏中下 | 空方占优，但尾盘有承接 |
| -0.8 ~ -0.5 | 低位收盘阴线 | 空方强势，可能有短下影线 |
| -1.0 ~ -0.8 | 光脚阴线（收盘接近最低） | 空方完全控盘，最强看跌信号 |

> **注意**：KSFT2 消除了绝对股价影响，适合跨股票横向比较。极端值（接近 ±1）通常预示趋势延续，接近 0 预示可能反转。与 KLEN 配合使用：窄幅（低 KLEN）下的极端 KSFT2 值比宽幅下的同类值信号更强。

### 信号解读

#### 10. OPEN0 — 开盘价相对收盘价

| 值范围 | 日内形态 | 市场信号 |
|--------|---------|----------|
| > 1.03 | 高开 3%+ 后低走 | 开盘情绪过热，盘中持续回落，可能利好出尽 |
| 1.01 ~ 1.03 | 小幅高开低走 | 开盘偏强但承接不足，多方后继乏力 |
| 0.99 ~ 1.01 | 平开（开收接近） | 多空均衡，当日无明显方向 |
| 0.97 ~ 0.99 | 小幅低开高走 | 开盘偏弱但盘中有买盘承接，空方力量衰减 |
| < 0.97 | 低开 3%+ 后高走 | 恐慌性低开后强势收复，可能利空出尽或主力吸筹 |

> **注意**：OPEN0 与 KMID 高度相关但**分母不同**——KMID 除以 `$open`，OPEN0 除以 `$close`。OPEN0 更强调"开盘价相对于最终收盘结果"的偏离程度，而 KMID 衡量的是从开盘出发的涨跌幅。在尾盘收盘价被大单拉升/砸盘的场景下，OPEN0 比 KMID 更能捕捉这种异常。

#### 11. HIGH0 — 最高价相对收盘价（盘中冲高程度）

| 值范围 | 日内形态 | 市场信号 |
|--------|---------|----------|
| > 1.05 | 盘中冲高 5%+ 后大幅回落 | 极强抛压，多方进攻完全失败，长上影线，看跌 |
| 1.03 ~ 1.05 | 冲高 3%~5% 后回落 | 上方阻力明显，冲高未能站稳 |
| 1.01 ~ 1.03 | 温和冲高 | 正常日内波动，上方有一定压力 |
| 1.00 ~ 1.01 | 最高价接近收盘价 | 光头或近光头（收盘即最高），多方强势控盘至收盘 |
| = 1.00 | 收盘即最高价 | 光头阳线，最强看涨信号，尾盘持续拉升 |

> **注意**：HIGH0 永远 ≥ 1（最高价不可能低于收盘价）。HIGH0 接近 1 说明收盘就是当天最高点，是极强的多头信号；HIGH0 越大说明冲高回落越严重。与 KUP（上影线长度）的区别：KUP 衡量上影线的绝对长度，HIGH0 衡量的是最高价整体相对收盘价的偏离。

#### 12. LOW0 — 最低价相对收盘价（盘中下探程度）

| 值范围 | 日内形态 | 市场信号 |
|--------|---------|----------|
| = 1.00 | 收盘即最低价 | 光脚阴线，最强看跌信号，空方控盘至收盘 |
| 0.99 ~ 1.00 | 最低价接近收盘价 | 光脚或近光脚，空方强势，下方无承接 |
| 0.97 ~ 0.99 | 温和下探 | 正常日内波动，下方有一定支撑 |
| 0.95 ~ 0.97 | 下探 3%~5% 后反弹 | 下方支撑明显，空方打压后买盘介入 |
| < 0.95 | 盘中暴跌 5%+ 后大幅反弹 | 极强承接，空方进攻被拉回，长下影线，看涨 |

> **注意**：LOW0 永远 ≤ 1（最低价不可能高于收盘价）。LOW0 接近 1 说明收盘就是当天最低点，是最强的空头信号；LOW0 越小说明下探后反弹越猛。与 KLOW（下影线长度）的关系类似 HIGH0 与 KUP 的关系。

#### 13. VWAP0 — 均价相对收盘价（尾盘强弱）

| 值范围 | 日内形态 | 市场信号 |
|--------|---------|----------|
| > 1.02 | 均价远高于收盘 | 尾盘明显走弱，全天大部分时间价格高于收盘，尾盘跳水 |
| 1.005 ~ 1.02 | 均价略高于收盘 | 尾盘偏弱，收盘未能维持在全天均价之上 |
| 0.995 ~ 1.005 | 均价约等于收盘 | 全天交易均匀，尾盘无明显异动 |
| 0.98 ~ 0.995 | 均价略低于收盘 | 尾盘偏强，收盘价被拉升至均价之上 |
| < 0.98 | 均价远低于收盘 | 尾盘明显走强，收盘大幅拉升，可能有资金尾盘抢筹 |

> **注意**：VWAP（Volume Weighted Average Price）是全天成交量加权均价，代表"市场真实的平均持仓成本"。VWAP0 本质上在回答：**收盘价是在全天均价之上还是之下？** 收盘 > VWAP（VWAP0 < 1）说明尾盘走强，持仓者整体浮盈；收盘 < VWAP（VWAP0 > 1）说明尾盘走弱，持仓者整体浮亏。
>
> VWAP0 是这 4 个因子中**信息量最丰富**的一个，因为它融合了成交量信息，而不仅仅是价格。机构交易者常以 VWAP 作为基准价格（benchmark），收盘价相对 VWAP 的位置是衡量资金流向的重要指标。

#### 四个因子的协同解读

| 组合模式 | 因子特征 | K线形态 | 市场信号 |
|----------|---------|---------|----------|
| **高开低走** | OPEN0 > 1, HIGH0 ≈ OPEN0, LOW0 < 1, VWAP0 > 1 | 高开大阴线 | 利好出尽或主力借高开出货 |
| **低开高走** | OPEN0 < 1, HIGH0 > 1, LOW0 ≈ OPEN0, VWAP0 < 1 | 低开大阳线 | 利空出尽或主力借低开吸筹 |
| **冲高回落** | OPEN0 ≈ 1, HIGH0 >> 1, LOW0 ≤ 1, VWAP0 > 1 | 长上影线 | 上方抛压沉重，多方进攻失败 |
| **探底回升** | OPEN0 ≈ 1, HIGH0 ≥ 1, LOW0 << 1, VWAP0 < 1 | 长下影线 | 下方支撑强劲，空方打压被拉回 |
| **光头光脚阳** | OPEN0 < 1, HIGH0 ≈ 1, LOW0 < 1, VWAP0 < 1 | 光头光脚阳线 | 多方完全控盘，最强看涨 |
| **光头光脚阴** | OPEN0 > 1, HIGH0 > 1, LOW0 ≈ 1, VWAP0 > 1 | 光头光脚阴线 | 空方完全控盘，最强看跌 |
| **十字星** | OPEN0 ≈ 1, HIGH0 略>1, LOW0 略<1, VWAP0 ≈ 1 | 十字星 | 多空均衡，变盘信号 |

> **与 K 线形态因子的关系**：价格截面因子（OPEN0/HIGH0/LOW0/VWAP0）和 K 线形态因子（KMID/KLEN/KUP 等）描述的是**同一根 K 线**，但视角不同。K 线因子以开盘价为基准（分母多为 `$open`），价格截面因子以收盘价为基准（分母为 `$close`）。两者互补使用效果更好：K 线因子告诉你"从开盘看发生了什么"，截面因子告诉你"相对于最终结果，过程中的偏离有多大"。

## 三、滚动窗口因子（145个 = 29种算子 × 5个窗口）

以下按算子分类，展示公式和含义。每个算子都在 5、10、20、30、60 日窗口上计算。

### 3.1 ROC — 价格变动率（动量因子）

**直觉**：N 天前的价格和今天相比，涨了还是跌了。

```
ROC_N = Ref($close, N) / $close
```

| N=5 | N=10 | N=20 | N=30 | N=60 |
|:---:|:----:|:----:|:----:|:----:|
| 1.0061 | 1.0173 | 1.0630 | 1.0691 | 1.0187 |

> ROC > 1 = 近期下跌（N天前比现在贵），ROC < 1 = 近期上涨

### 信号解读

> **反直觉注意**：ROC 的分子是过去价格、分母是今天价格，所以 ROC > 1 表示**下跌**（过去比现在贵），ROC < 1 表示**上涨**（过去比现在便宜），与常见涨跌幅方向相反。

| ROC 值 | 等价涨跌幅 | 含义 |
|:------:|:---------:|------|
| 0.95 | +5.3% | N 天前比今天便宜 5%，近期**大涨** |
| 0.98 | +2.0% | 近期温和上涨 |
| 1.00 | 0% | 价格没变 |
| 1.02 | -2.0% | 近期温和下跌 |
| 1.05 | -4.8% | 近期明显下跌 |

#### 多窗口协同模式

| 模式 | 特征 | 信号 |
|------|------|------|
| **全面上涨** | 所有 ROC < 1 | 各周期都在涨，强趋势 |
| **全面下跌** | 所有 ROC > 1 | 各周期都在跌，弱趋势 |
| **短期反转** | ROC5 < 1 但 ROC60 > 1 | 长期下跌中出现短期反弹，可能见底 |
| **短涨长跌** | ROC5 < 1 但 ROC60 > 1 | 下跌趋势中的反弹，需警惕是否为诱多 |
| **短跌长涨** | ROC5 > 1 但 ROC60 < 1 | 上涨趋势中的短期回调，可能是买点 |

> **局限性**：ROC 只看起点和终点，不看中间过程。若 N 天前恰好是涨跌停，ROC 会失真。需配合 RSQR（趋势流畅度）和 STD（波动率）来过滤噪音。
>
> **与 MA 的区别**：ROC 用 N 天前**单日**价格，MA 用 N 天**均价**。MA 更平滑抗噪，ROC 对短期爆发力更敏感。

### 3.2 MA — 移动均线（趋势因子）

**直觉**：N 日均价与今日收盘价的关系，等价于常见的"收盘价站上/跌破 MA-N"。衡量趋势方向。

```
MA_N = Mean($close, N) / $close
```

| MA5 | MA10 | MA20 | MA30 | MA60 |
|:---:|:----:|:----:|:----:|:----:|
| 1.0026 | 1.0078 | 1.0256 | 1.0417 | 1.0650 |

> MA < 1 = 收盘价在均线上方，MA > 1 = 收盘价在均线下方

### 信号解读

> **同样反直觉**：与 ROC 相同，分子是“历史”、分母是“今天”。MA > 1 = 均价高于现价 = 收盘价在均线**下方**（弱势）；MA < 1 = 均价低于现价 = 收盘价在均线**上方**（强势）。

| MA 值 | 等价偏离度 | 含义 |
|:-----:|:---------:|------|
| 0.95 | 收盘高于均线 5.3% | 收盘价远在均线上方，强势 |
| 0.98 | 收盘高于均线 2.0% | 收盘价在均线上方，温和偏多 |
| 1.00 | 收盘 ≈ 均线 | 正好踩在均线上，支撑/阻力测试 |
| 1.02 | 收盘低于均线 2.0% | 收盘价在均线下方，温和偏空 |
| 1.05 | 收盘低于均线 4.8% | 收盘价远离均线下方，弱势 |

#### 多窗口协同模式（均线排列）

| 模式 | 特征 | 信号 |
|------|------|------|
| **多头排列** | MA5 < MA10 < MA20 < MA60（都 < 1） | 短中长期均线全部在现价下方，强势上涨趋势 |
| **空头排列** | MA5 > MA10 > MA20 > MA60（都 > 1） | 短中长期均线全部在现价上方，弱势下跌趋势 |
| **金叉信号** | MA5 从 > MA10 变为 < MA10 | 短期均线上穿长期均线，趋势转多 |
| **死叉信号** | MA5 从 < MA10 变为 > MA10 | 短期均线下穿长期均线，趋势转空 |
| **均线粘合** | MA5 ≈ MA10 ≈ MA20 ≈ 1 | 所有均线聚拢，变盘在即（方向选择） |
| **短多长空** | MA5 < 1 但 MA60 > 1 | 短期反弹但长期仍在均线下方，可能是反弹而非反转 |

> **与 ROC 的区别**：ROC 捕捉“N 天涨跌幅”（动量大小），对单日异常敏感；MA 衡量“收盘价相对均线位置”（趋势方向），因均值平滑而更稳定。ROC 适合动量捕捉，MA 适合趋势判断和支撑阻力识别。

### 3.3 STD — 历史波动率（波动因子）

**直觉**：过去 N 日收盘价的标准差，衡量价格波动剧烈程度。除以当日收盘价消除量纲。

```
STD_N = Std($close, N) / $close
```

| STD5 | STD10 | STD20 | STD30 | STD60 |
|:----:|:-----:|:-----:|:-----:|:-----:|
| 0.0044 | 0.0075 | 0.0220 | 0.0300 | 0.0422 |

> 值越大 = 近期波动越剧烈

### 信号解读

| STD 值 | 等价日波动 | 波动等级 | 典型场景 |
|:------:|:---------:|:-------:|----------|
| < 0.005 | < 0.5% | 极低波动 | 缩量横盘、窄幅整理（变盘前兆） |
| 0.005 ~ 0.015 | 0.5% ~ 1.5% | 正常波动 | 日常交易，无异常 |
| 0.015 ~ 0.030 | 1.5% ~ 3.0% | 中等波动 | 板块活跃、有消息刺激 |
| 0.030 ~ 0.050 | 3.0% ~ 5.0% | 高波动 | 热点题材、多空分歧大 |
| > 0.050 | > 5.0% | 极端波动 | 连续涨跌停、重大事件驱动 |

#### 多窗口协同模式

| 模式 | 特征 | 信号 |
|------|------|------|
| **低波收敛** | 所有 STD 都小，STD5 < STD60 | 波动收窄，变盘在即（挤压弹簧） |
| **波动爆发** | STD5 >> STD60 | 近期突然放量波动，有大事发生 |
| **持续高波动** | 所有 STD 都大 | 长期活跃股，适合日内/波段 |
| **波动率递减** | STD5 < STD10 < STD20 < STD60 | 曾经活跃但正在降温，回归平静 |
| **短高长低** | STD5 大但 STD60 小 | 平时稳定但近期异常，可能是事件驱动 |

> **与 KLEN 的区别**：KLEN 衡量单日振幅（当天最高-最低），STD 衡量多日收盘价的离散程度。一根长十字星（高开低走但收平），KLEN 很大但 STD 不大；连续 5 天各涨 1%（每天振幅很小），KLEN 不大但 STD 较大。
>
> **与 MA 的配合**：STD + MA 组合即布林带思想。MA > 1 + STD 小 = 底部横盘蓄势；MA < 1 + STD 大 = 强势主升浪；MA > 1 + STD 大 = 恐慌性抛售；MA < 1 + STD 小 = 高位横盘可能变盘。

### 3.4 BETA — 趋势斜率（动量因子）

**直觉**：过去 N 天收盘价做线性回归，斜率代表"每天涨多少"。类似 β，但仅对价格自身回归。

```
BETA_N = Slope($close, N) / $close
```

| BETA5 | BETA10 | BETA20 | BETA30 | BETA60 |
|:-----:|:------:|:------:|:------:|:------:|
| 0.0004 | -0.0020 | -0.0035 | -0.0032 | -0.0018 |

> 正 = 上升趋势，负 = 下降趋势

### 信号解读

> BETA 衡量的是**每天涨/跌多少百分比**（线性回归斜率归一化），利用窗口内所有数据点，比 ROC（只看首尾）更能反映过程中的趋势方向。

| BETA 值 | 等价每日涨跌 | 含义 |
|:------:|:---------:|------|
| +0.010 | 每天涨 1.0% | 快速上涨通道，20 天涨约 22% |
| +0.005 | 每天涨 0.5% | 温和上涨 |
| +0.001 | 每天涨 0.1% | 微弱上行 |
| 0 | 不涨不跌 | 横盘 |
| -0.001 | 每天跌 0.1% | 微弱下行 |
| -0.005 | 每天跌 0.5% | 温和下跌 |
| -0.010 | 每天跌 1.0% | 快速下跌通道 |

#### 多窗口协同模式

| 模式 | 特征 | 信号 |
|------|------|------|
| **加速上涨** | BETA5 > BETA60 > 0 | 短期涨速超过长期，趋势加速 |
| **减速上涨** | 0 < BETA5 < BETA60 | 还在涨但速度放缓，可能见顶 |
| **加速下跌** | BETA5 < BETA60 < 0 | 短期跌速超过长期，恐慌加剧 |
| **减速下跌** | BETA60 < BETA5 < 0 | 还在跌但速度放缓，可能见底 |
| **短多长空** | BETA5 > 0 > BETA60 | 长期下跌中出现短期反弹 |
| **短空长多** | BETA5 < 0 < BETA60 | 长期上涨中出现短期回调 |

> **局限性**：大涨/大跌等异常点会扭曲回归线，导致斜率失真。必须配合 RSQR（拟合度）判断可信度。
>
> **与 ROC/MA 的区别**：ROC 只看首尾（“走了多远”），MA 看偏离均线（“站在哪边”），BETA 利用所有数据点拟合（“走得多快、什么方向”）。

### 3.5 RSQR — 趋势拟合度 R²（趋势因子）

**直觉**：过去 N 天价格的 R²，衡量"价格走直线的程度"。接近 1 = 趋势很流畅，接近 0 = 震荡无序。

```
RSQR_N = Rsquare($close, N)
```

| RSQR5 | RSQR10 | RSQR20 | RSQR30 | RSQR60 |
|:-----:|:------:|:------:|:------:|:------:|
| 0.0213 | 0.6531 | 0.9049 | 0.8915 | 0.5341 |

> 高 RSQR + 正 BETA = 稳健上涨趋势；低 RSQR = 方向不明

### 信号解读

> RSQR 衡量“价格走直线的程度”，即线性回归的 R²。R² = 1 - (残差平方和/总变异平方和)，接近 1 表示数据点贴合趋势线，接近 0 表示杂乱无章。

| RSQR 值 | 拟合程度 | 图形直觉 |
|:------:|:-------:|----------|
| > 0.90 | 极佳 | 数据点几乎落在一条直线上 |
| 0.70 ~ 0.90 | 较好 | 大致沿直线，偶有偏离 |
| 0.50 ~ 0.70 | 中等 | 有方向感但散乱 |
| 0.20 ~ 0.50 | 较差 | 趋势很弱，大部分是噪音 |
| < 0.20 | 极差 | 接近随机游走，无趋势 |

#### BETA + RSQR 配对解读

> RSQR 是 BETA 的“可信度检测器”——BETA 告诉你方向，RSQR 告诉你这个方向可信吗。

| 组合 | 含义 |
|------|------|
| BETA > 0 + RSQR > 0.8 | 可信的上涨趋势（走得稳、方向明确） |
| BETA > 0 + RSQR < 0.2 | 不可信的“上涨”（中间上蹕下跳，只是首尾碰巧涨了） |
| BETA < 0 + RSQR > 0.8 | 可信的下跌趋势 |
| BETA ≈ 0 + RSQR < 0.2 | 横盘震荡，无方向 |

#### 多窗口协同模式

| 模式 | 特征 | 信号 |
|------|------|------|
| **长短期都流畅** | RSQR5、RSQR60 都高 | 各周期都有趋势，走势非常明确 |
| **长流畅短混乱** | RSQR60 高但 RSQR5 低 | 长期趋势清晰，但短期扰动 → 可能正在变盘 |
| **短流畅长混乱** | RSQR5 高但 RSQR60 低 | 短期有方向但长期无趋势 → 可能只是短期脉冲 |
| **全周期混乱** | 所有 RSQR 都低 | 纯震荡，无趋势 → 不适合趋势策略 |

### 3.6 RESI — 趋势残差（反转因子）

**直觉**：最近一天价格偏离趋势线的程度。残差大 = 价格突然偏离原有趋势。

```
RESI_N = Resi($close, N) / $close
```

| RESI5 | RESI10 | RESI20 | RESI30 | RESI60 |
|:-----:|:------:|:------:|:------:|:------:|
| -0.0035 | 0.0012 | 0.0080 | 0.0050 | -0.0128 |

> 正值 = 今天高于趋势线（可能超买），负值 = 低于趋势线（可能超卖）

### 信号解读

> RESI 只看**最后一天（今天）**的残差：实际收盘价 - 趋势线在今天位置的值，再除以收盘价归一化。正值 = 今天站在趋势线上方，负值 = 跌破趋势线。

| RESI 值 | 偏离程度 | 含义 |
|:------:|:-------:|------|
| > +0.02 | 远高于趋势线 | 短期加速偏离，可能超买 |
| +0.005 ~ +0.02 | 略高于趋势线 | 走势健康，略强于趋势 |
| -0.005 ~ +0.005 | 贴合趋势线 | 价格紧跟趋势，无异常 |
| -0.02 ~ -0.005 | 略低于趋势线 | 走势略弱，可能回调 |
| < -0.02 | 远低于趋势线 | 短期加速偏离，可能超卖 |

#### BETA + RSQR + RESI 三因子联动

> 这三个因子来自**同一条回归线**，各回答一个维度：BETA = 趋势方向（路往哪走），RSQR = 趋势可信度（路修得直不直），RESI = 今天偏离多少（车偏离道路多远）。

| 组合 | 含义 | 操作建议 |
|------|------|----------|
| BETA>0 + RSQR高 + RESI≈0 | 稳健上涨中，价格贴合趋势 | 持有/追多 |
| BETA>0 + RSQR高 + RESI大正 | 上涨趋势中突然加速偏离 | 警惕短期超买回调 |
| BETA<0 + RSQR高 + RESI大负 | 下跌趋势中突然加速偏离 | 警惕短期超卖反弹 |
| BETA<0 + RSQR高 + RESI转正 | 下跌趋势清晰，但今天站上趋势线 | 可能见底反转信号 |
| 任意BETA + RSQR低 | 无趋势，斜率不可信 | 不适合趋势策略 |

#### 多窗口协同模式

| 模式 | 特征 | 信号 |
|------|------|------|
| **全周期正向偏离** | 所有 RESI > 0 | 各周期都在趋势线上方，强势超买 |
| **全周期负向偏离** | 所有 RESI < 0 | 各周期都在趋势线下方，弱势超卖 |
| **短正长负** | RESI5 > 0 但 RESI60 < 0 | 短期反弹但仍在长期趋势线下方 → 底部反转试探 |
| **短负长正** | RESI5 < 0 但 RESI60 > 0 | 短期回调但长期趋势线上方 → 上涨中正常回踩 |

### 3.7 MAX — N 日最高价（趋势因子/价格位置）

**直觉**：过去 N 天最高价相对于今天收盘价。

```
MAX_N = Max($high, N) / $close
```

| MAX5 | MAX10 | MAX20 | MAX30 | MAX60 |
|:----:|:-----:|:-----:|:-----:|:-----:|
| 1.0132 | 1.0254 | 1.0681 | 1.1159 | 1.2230 |

> 接近 1 = 今天接近 N 日最高点

### 信号解读

> MAX 永远 ≥ 1（最高价不可能低于收盘价）。值越接近 1 说明今天越接近 N 日高点。

| MAX 值 | 距高点 | 含义 |
|:------:|:-------:|------|
| = 1.00 | 0% | 今天就是 N 日最高点（创新高） |
| 1.01 ~ 1.02 | 1%~2% | 非常接近高点，强势 |
| 1.02 ~ 1.05 | 2%~5% | 离高点不远 |
| 1.05 ~ 1.10 | 5%~10% | 明显偏离高点 |
| > 1.10 | 10%+ | 远离高点，处于低位 |

### 3.8 MIN — N 日最低价（趋势因子/价格位置）

**直觉**：过去 N 天最低价相对于今天收盘价。

```
MIN_N = Min($low, N) / $close
```

| MIN5 | MIN10 | MIN20 | MIN30 | MIN60 |
|:----:|:-----:|:-----:|:-----:|:-----:|
| 0.9959 | 0.9959 | 0.9959 | 0.9959 | 0.9959 |

> 接近 1 = 今天接近 N 日最低点

### 信号解读

> MIN 永远 ≤ 1（最低价不可能高于收盘价）。值越接近 1 说明今天越接近 N 日低点。

| MIN 值 | 距低点 | 含义 |
|:------:|:-------:|------|
| = 1.00 | 0% | 今天就是 N 日最低点（创新低） |
| 0.98 ~ 1.00 | 0%~2% | 非常接近低点，弱势 |
| 0.95 ~ 0.98 | 2%~5% | 离低点不远 |
| 0.90 ~ 0.95 | 5%~10% | 明显偏离低点 |
| < 0.90 | 10%+ | 远离低点，处于高位 |

#### MAX + MIN 组合解读

| 组合模式 | 特征 | 信号 |
|----------|------|------|
| **逼近新高** | MAX ≈ 1 且 MIN 远小于 1 | 价格在 N 日顶部，突破信号 |
| **逼近新低** | MIN ≈ 1 且 MAX 远大于 1 | 价格在 N 日底部，破位风险 |
| **区间中部** | MAX 和 MIN 偏离程度相当 | 方向不明 |
| **区间收窄** | MAX 和 MIN 都接近 1 | 窄幅震荡，变盘前兆 |

> **与 RSV 的区别**：RSV 把 Max 和 Min 合成一个数（价格在区间中的百分比），MAX 和 MIN 各自独立提供上方压力和下方支撑的信息。

### 3.9 QTLU — 上分位数 80%（波动因子）

**直觉**：过去 N 天收盘价的 80% 分位数。80% 的收盘价低于此值。

```
QTLU_N = Quantile($close, N, 0.8) / $close
```

| QTLU5 | QTLU10 | QTLU20 | QTLU30 | QTLU60 |
|:-----:|:------:|:------:|:------:|:------:|
| 1.0037 | 1.0132 | 1.0520 | 1.0673 | 1.0885 |

### 信号解读

> QTLU 衡量过去 N 天收盘价的 80% 分位数相对于今天的位置。QTLU > 1 表示 80% 的历史收盘价高于今天，今天偏低。

| QTLU 值 | 含义 |
|:------:|------|
| ≈ 1.00 | 今天就在 80% 分位附近，处于近期偏高位置 |
| 1.02 ~ 1.05 | 80% 分位比今天高 2%~5%，今天处于中偏低位置 |
| > 1.05 | 80% 分位远高于今天，今天处于分布底部 |

### 3.10 QTLD — 下分位数 20%（波动因子）

**直觉**：过去 N 天收盘价的 20% 分位数。20% 的收盘价低于此值。

```
QTLD_N = Quantile($close, N, 0.2) / $close
```

| QTLD5 | QTLD10 | QTLD20 | QTLD30 | QTLD60 |
|:-----:|:------:|:------:|:------:|:------:|
| 0.9998 | 1.0016 | 1.0053 | 1.0098 | 1.0230 |

### 信号解读

> QTLD 衡量过去 N 天收盘价的 20% 分位数相对于今天的位置。QTLD > 1 表示连底部 20% 的历史收盘价都高于今天，今天非常低。

| QTLD 值 | 含义 |
|:------:|------|
| < 0.98 | 20% 分位低于今天 2%+，今天处于分布偏高位置 |
| 0.98 ~ 1.00 | 今天接近 20% 分位，偏低 |
| > 1.00 | 连底部 20% 分位都高于今天，今天非常低 |

#### QTLU + QTLD 组合解读

> QTLU 和 QTLD 框出了过去 N 天价格的**中间 60% 区间**（20%~80% 分位）。两者之差（QTLU - QTLD）近似衡量价格分布的离散程度，与 STD 类似但不受极端值影响（更鲁棒）。

| 组合模式 | 特征 | 含义 |
|----------|------|------|
| 今天在区间下方 | QTLD > 1 | 今天处于 N 日分布底部，弱势 |
| 今天在区间上方 | QTLU ≈ 1 | 今天处于 N 日分布顶部，强势 |
| 今天在区间内 | QTLD < 1 且 QTLU > 1 | 今天处于正常范围中部 |
| 区间收窄 | QTLU 和 QTLD 都接近 1 | 价格分布集中，波动小，变盘前兆 |

### 3.11 RANK — 价格百分位（反转因子/超买超卖）

**直觉**：今天收盘价在过去 N 天的排名（0~1）。0.9 = 今收比 90% 的历史收盘价都高。

```
RANK_N = Rank($close, N)
```

| RANK5 | RANK10 | RANK20 | RANK30 | RANK60 |
|:-----:|:------:|:------:|:------:|:------:|
| 0.4000 | 0.2000 | 0.1000 | 0.0667 | 0.0333 |

> 接近 1 = 近期高位，接近 0 = 近期低位。0.5 = 居中。

### 信号解读

> RANK 只看排名，不看具体价格，因此不受极端值影响（比 MAX/MIN/RSV 更鲁棒）。

| RANK 值 | 百分位 | 含义 |
|:------:|:-----:|------|
| > 0.8 | 前 20% | 近期高位，可能超买 |
| 0.6 ~ 0.8 | 偏高 | 偏强但还未极端 |
| 0.4 ~ 0.6 | 中间 | 居中，无明确信号 |
| 0.2 ~ 0.4 | 偏低 | 偏弱 |
| < 0.2 | 后 20% | 近期低位，可能超卖 |

#### 多窗口协同模式

| 模式 | 特征 | 信号 |
|------|------|------|
| **全面高位** | 所有 RANK > 0.8 | 各周期都在高位，强超买 |
| **全面低位** | 所有 RANK < 0.2 | 各周期都在低位，强超卖 |
| **短高长低** | RANK5 > 0.8 但 RANK60 < 0.2 | 长期下跌中短期反弹（可能只是脉冲） |
| **短低长高** | RANK5 < 0.2 但 RANK60 > 0.8 | 长期上涨中短期回调（可能是买点） |

> **与 RSV 的区别**：RSV = (close-Min)/(Max-Min)，受极端最高/最低价影响；RANK 只看排名，即使有一天涨停也不会扭曲其他天的排名。RSV 反映“在箱体中的位置”，RANK 反映“在历史中的百分位”。


### 3.12 RSV — 相对强弱值（反转因子/KDJ 基础）

**直觉**：K 线图中 KDJ 的 RSV 值。今天收盘价在 N 日最高-最低区间中的位置。

```
RSV_N = ($close - Min($low, N)) / (Max($high, N) - Min($low, N) + 1e-12)
```

| RSV5 | RSV10 | RSV20 | RSV30 | RSV60 |
|:----:|:-----:|:-----:|:-----:|:-----:|
| 0.2353 | 0.1379 | 0.0563 | 0.0339 | 0.0179 |

> 0 = 收在 N 日最低，1 = 收在 N 日最高。KDJ 中的 K/D/J 线由此衍生。

### 信号解读

> RSV 就是传统技术分析中 KDJ 指标的原始值，衡量今天收盘价在 N 日“最高-最低”箱体中的百分比位置。

| RSV 值 | 箱体位置 | 含义 |
|:------:|:-------:|------|
| > 0.8 | 上半区顶部 | 强势，接近 N 日高点，可能超买 |
| 0.5 ~ 0.8 | 上半区 | 偏强，在区间中上部 |
| = 0.5 | 正中 | 居中，无方向 |
| 0.2 ~ 0.5 | 下半区 | 偏弱，在区间中下部 |
| < 0.2 | 下半区底部 | 弱势，接近 N 日低点，可能超卖 |

> **与 RANK 的区别**：RSV 受极端高/低价影响（涨停拉大箱体，压缩 RSV）；RANK 只看排名，不受极端值影响。RSV 反映“在箱体中的位置”，RANK 反映“在历史中的百分位”。
>
> **与 KDJ 的关系**：K = RSV 的 3 日平滑，D = K 的 3 日平滑，J = 3K - 2D。Alpha158 直接用 RSV，让模型自己学习如何平滑。

### 3.13 IMAX — 距前高天数 / Aroon 上轨（趋势因子/突破信号）

**直觉**：Aroon 指标的上轨。当前距过去 N 天内最高价那天的天数/窗口。

```
IMAX_N = IdxMax($high, N) / N
```

| IMAX5 | IMAX10 | IMAX20 | IMAX30 | IMAX60 |
|:-----:|:------:|:------:|:------:|:------:|
| 1.0000 | 0.1000 | 0.0500 | 0.1667 | 0.1000 |

> 接近 0 = 刚创 N 日新高，接近 1 = 很久没创新高

### 信号解读

> IMAX = 距 N 日最高价的天数 ÷ N。IMAX ≈ 0 表示刚创新高（强势），IMAX ≈ 1 表示高点在窗口最远端（之后一路下跌）。

| IMAX 值 | 高点距今 | 含义 |
|:------:|:-------:|------|
| ≈ 0 | 今天或昨天 | 刚创 N 日新高，强势突破 |
| 0 ~ 0.2 | 很近 | 近期仍在创新高附近 |
| 0.2 ~ 0.5 | 中等 | 高点已过去一段时间 |
| 0.5 ~ 0.8 | 较远 | 很久没创新高，上涨动能减弱 |
| ≈ 1.0 | 最远 | 高点在窗口最远端，之后一路下跌 |

### 3.14 IMIN — 距前低天数 / Aroon 下轨（趋势因子/见底信号）

**直觉**：Aroon 指标的下轨。当前距过去 N 天内最低价那天的天数/窗口。

```
IMIN_N = IdxMin($low, N) / N
```

| IMIN5 | IMIN10 | IMIN20 | IMIN30 | IMIN60 |
|:-----:|:------:|:------:|:------:|:------:|
| 0.4000 | 0.7000 | 0.8500 | 0.9000 | 0.9500 |

> 接近 0 = 刚创 N 日新低

### 信号解读

> IMIN 与 IMAX 对称：距 N 日最低价的天数 ÷ N。IMIN ≈ 0 表示刚创新低（弱势），IMIN ≈ 1 表示低点在窗口最远端（之后一路反弹）。

| IMIN 值 | 低点距今 | 含义 |
|:------:|:-------:|------|
| ≈ 0 | 今天或昨天 | 刚创 N 日新低，弱势破位 |
| 0 ~ 0.2 | 很近 | 近期仍在创新低附近 |
| 0.2 ~ 0.5 | 中等 | 低点已过去一段时间 |
| 0.5 ~ 0.8 | 较远 | 很久没创新低，下跌动能减弱 |
| ≈ 1.0 | 最远 | 低点在窗口最远端，之后一路反弹 |

#### IMAX + IMIN 组合（Aroon 指标）

| 组合模式 | 特征 | 信号 |
|----------|------|------|
| **强势突破** | IMAX ≈ 0 且 IMIN 大 | 刚创新高且很久没创新低 → 上升趋势 |
| **弱势破位** | IMIN ≈ 0 且 IMAX 大 | 刚创新低且很久没创新高 → 下降趋势 |
| **高位震荡** | IMAX 和 IMIN 都小 | 近期既创新高又创新低 → 剧烈震荡 |
| **死水一潭** | IMAX 和 IMIN 都大 | 很久没创新高也没创新低 → 横盘整理 |

### 3.15 IMXD — 高低点时间差（趋势形态因子）

**直觉**：N 日内最高点和最低点之间的时间差。正值 = 高点发生在低点之后（先跌后涨）。

```
IMXD_N = (IdxMax($high, N) - IdxMin($low, N)) / N
```

| IMXD5 | IMXD10 | IMXD20 | IMXD30 | IMXD60 |
|:-----:|:------:|:------:|:------:|:------:|
| 0.6000 | -0.6000 | -0.8000 | -0.7333 | -0.8500 |

> 正值 = 上升后见顶，负值 = 下跌后见底

### 信号解读

> IMXD = (高点距今天数 - 低点距今天数) / N。正值 = 高点在低点之后（先跌后涨，V 形），负值 = 高点在低点之前（先涨后跌，倒 V 形）。

| IMXD 值 | 含义 | 时间顺序 |
|:------:|------|----------|
| > +0.5 | 强正值 | 先见底再冲高，V 形反弹 |
| +0.2 ~ +0.5 | 弱正值 | 低点在前、高点在后，偏上升形态 |
| -0.2 ~ +0.2 | 接近零 | 高低点几乎同时出现，罕见 |
| -0.5 ~ -0.2 | 弱负值 | 高点在前、低点在后，偏下降形态 |
| < -0.5 | 强负值 | 先冲高再见底，倒 V 形下跌 |

#### IMAX + IMIN + IMXD 三因子联动

| 组合 | 含义 |
|------|------|
| IMAX≈0 + IMIN大 + IMXD>0 | 刚创新高、低点在远处、先低后高 → **强上升趋势** |
| IMAX大 + IMIN≈0 + IMXD<0 | 高点在远处、刚创新低、先高后低 → **强下降趋势** |
| IMAX≈0 + IMIN≈0 | 近期既创新高又创新低 → **剧烈震荡** |
| IMAX大 + IMIN大 | 高低点都在远处 → **横盘整理** |

### 3.16 CORR — 价量相关系数（量价配合因子）

**直觉**：过去 N 天收盘价和成交量的相关性。正相关 = 价涨量增，负相关 = 价涨量缩。

```
CORR_N = Corr($close, Log($volume + 1), N)
```

| CORR5 | CORR10 | CORR20 | CORR30 | CORR60 |
|:-----:|:------:|:------:|:------:|:------:|
| 0.9276 | 0.2250 | -0.1627 | -0.0681 | 0.6221 |

> 正 = 量价配合良好，负 = 量价背离

### 信号解读

> CORR 衡量过去 N 天收盘价与成交量的相关系数。正相关 = 价涨量增（健康），负相关 = 价涨量缩（背离）。

| CORR 值 | 量价关系 | 含义 |
|:------:|:-------:|------|
| > +0.5 | 强正相关 | 价涨量增、价跌量缩，量价配合良好（健康趋势） |
| +0.2 ~ +0.5 | 弱正相关 | 量价基本同向，但不强烈 |
| -0.2 ~ +0.2 | 无相关 | 价量无关，市场无序 |
| -0.5 ~ -0.2 | 弱负相关 | 量价略有背离 |
| < -0.5 | 强负相关 | 价涨量缩或价跌量增，量价背离（趋势不可信） |

> **CORR 与 CORD 的区别**：CORR 用原始价格和原始成交量，衡量“价格水平和量能水平”的关系；CORD 用日收益率和成交量变化率，衡量“涨跌方向和放缩量方向”的关系，更精细。

### 3.17 CORD — 收益率-量比相关系数（量价配合因子）

**直觉**：日收益率和日成交量变化率的相关性。衡量"涨的时候是否放量"。

```
CORD_N = Corr($close / Ref($close, 1), Log($volume / Ref($volume, 1) + 1), N)
```

| CORD5 | CORD10 | CORD20 | CORD30 | CORD60 |
|:-----:|:------:|:------:|:------:|:------:|
| 0.9524 | 0.7987 | 0.5473 | 0.5773 | 0.5078 |

> 正 = 放量上涨/缩量下跌（健康），负 = 上涨缩量（异常）

### 信号解读

> CORD 衡量过去 N 天日收益率与日成交量变化率的相关系数。正 = 涨时放量跌时缩量（健康资金流），负 = 涨时缩量（异常）。

| CORD 值 | 量价关系 | 含义 |
|:------:|:-------:|------|
| > +0.5 | 强正相关 | 涨时放量、跌时缩量，资金流向健康 |
| +0.2 ~ +0.5 | 弱正相关 | 量价配合基本合理 |
| -0.2 ~ +0.2 | 无相关 | 涨跌与放缩量无关 |
| < -0.2 | 负相关 | 涨时缩量或跌时放量，资金流向异常 |

#### CORR + CORD 组合解读

| 组合 | 含义 |
|------|------|
| CORR>0 + CORD>0 | 量价全面配合，趋势可信 |
| CORR>0 + CORD<0 | 价格水平与量能同向，但涨跌方向与放缩量不匹配 → 表面健康内在脆弱 |
| CORR<0 + CORD<0 | 量价全面背离，趋势不可信 |
| CORR<0 + CORD>0 | 价格水平与量能背离，但涨跌时放缩量合理 → 可能是洗盘 |

### 3.18 CNTP — 上涨天数占比（动量因子）

**直觉**：过去 N 天中收盘价上涨的天数比例。

```
CNTP_N = Mean($close > Ref($close, 1), N)
```

| CNTP5 | CNTP10 | CNTP20 | CNTP30 | CNTP60 |
|:-----:|:------:|:------:|:------:|:------:|
| 0.2000 | 0.2000 | 0.1500 | 0.2333 | 0.4167 |

> 0.5 = 涨跌各半

### 信号解读

> CNTP 衡量过去 N 天中收盘价上涨的天数比例。

| CNTP 值 | 含义 |
|:------:|------|
| > 0.7 | 上涨天数占绝大多数，强势 |
| 0.5 ~ 0.7 | 涨多跌少，偏多 |
| = 0.5 | 涨跌各半，平衡 |
| 0.3 ~ 0.5 | 跌多涨少，偏空 |
| < 0.3 | 下跌天数占绝大多数，弱势 |

### 3.19 CNTN — 下跌天数占比（动量因子）

```
CNTN_N = Mean($close < Ref($close, 1), N)
```

| CNTN5 | CNTN10 | CNTN20 | CNTN30 | CNTN60 |
|:-----:|:------:|:------:|:------:|:------:|
| 0.6000 | 0.7000 | 0.8000 | 0.7333 | 0.5667 |

> 注意：可能有平盘日，CNTP + CNTN ≤ 1

### 信号解读

> CNTN 与 CNTP 互补：衡量过去 N 天中收盘价下跌的天数比例。

| CNTN 值 | 含义 |
|:------:|------|
| > 0.7 | 下跌天数占绝大多数，弱势 |
| 0.5 ~ 0.7 | 跌多涨少，偏空 |
| = 0.5 | 涨跌各半 |
| 0.3 ~ 0.5 | 涨多跌少，偏多 |
| < 0.3 | 上涨天数占绝大多数，强势 |

### 3.20 CNTD — 涨跌天数净差（动量方向因子）

**直觉**：上涨天数减去下跌天数的净差值，反映方向性强度。

```
CNTD_N = Mean($close > Ref($close, 1), N) - Mean($close < Ref($close, 1), N)
```

| CNTD5 | CNTD10 | CNTD20 | CNTD30 | CNTD60 |
|:-----:|:------:|:------:|:------:|:------:|
| -0.4000 | -0.5000 | -0.6500 | -0.5000 | -0.1500 |

### 信号解读

> CNTD = CNTP - CNTN，上涨天数减去下跌天数的净差。正值 = 涨天多，负值 = 跌天多。

| CNTD 值 | 含义 |
|:------:|------|
| > +0.4 | 涨天远多于跌天，强势 |
| +0.1 ~ +0.4 | 涨天略多 |
| -0.1 ~ +0.1 | 涨跌平衡 |
| -0.4 ~ -0.1 | 跌天略多 |
| < -0.4 | 跌天远多于涨天，弱势 |

#### CNTP + CNTN + CNTD 的关系

> CNTP 看“涨天占比”，CNTN 看“跌天占比”，CNTD 看“净值”。三者与 SUMP/SUMN/SUMD 的关系：前者看**天数**（涨/跌了几天），后者看**幅度**（涨/跌了多少钱）。

### 3.21 SUMP — 累计涨幅占比 / 类 RSI（动量强度因子）

**直觉**：过去 N 天中，涨幅总和占涨跌绝对值总和的比例。等价于 RSI/100。

```
SUMP_N = Sum(max($close-Ref($close,1), 0), N) / Sum(|$close-Ref($close,1)|, N)
```

| SUMP5 | SUMP10 | SUMP20 | SUMP30 | SUMP60 |
|:-----:|:------:|:------:|:------:|:------:|
| 0.3929 | 0.3191 | 0.2281 | 0.3381 | 0.4865 |

> 0.5 = 涨跌幅度相等。> 0.5 = 涨幅大于跌幅（上涨有力）。

### 信号解读

> SUMP 等价于 RSI/100，衡量涨幅幅度占总波动的比例。

| SUMP 值 | 等价 RSI | 含义 |
|:------:|:------:|------|
| > 0.7 | RSI>70 | 涨幅远大于跌幅，强势/超买 |
| 0.5 ~ 0.7 | RSI 50~70 | 涨幅大于跌幅，偏多 |
| = 0.5 | RSI=50 | 涨跌幅度相等，平衡 |
| 0.3 ~ 0.5 | RSI 30~50 | 跌幅大于涨幅，偏空 |
| < 0.3 | RSI<30 | 跌幅远大于涨幅，弱势/超卖 |

### 3.22 SUMN — 累计跌幅占比（动量强度因子）

```
SUMN_N = Sum(max(Ref($close,1)-$close, 0), N) / Sum(|$close-Ref($close,1)|, N)
```

| SUMN5 | SUMN10 | SUMN20 | SUMN30 | SUMN60 |
|:-----:|:------:|:------:|:------:|:------:|
| 0.6071 | 0.6809 | 0.7719 | 0.6619 | 0.5135 |

> SUMN = 1 - SUMP，两者互为补集。

### 信号解读

> SUMN 与 SUMP 互补：衡量跌幅幅度占总波动的比例。SUMN = 1 - SUMP。

| SUMN 值 | 含义 |
|:------:|------|
| > 0.7 | 跌幅远大于涨幅，弱势 |
| 0.5 ~ 0.7 | 跌幅大于涨幅，偏空 |
| = 0.5 | 涨跌幅度相等 |
| 0.3 ~ 0.5 | 涨幅大于跌幅，偏多 |
| < 0.3 | 涨幅远大于跌幅，强势 |

### 3.23 SUMD — 涨跌幅净差（动量强度因子）

```
SUMD_N = SUMP_N - SUMN_N
```

| SUMD5 | SUMD10 | SUMD20 | SUMD30 | SUMD60 |
|:-----:|:------:|:------:|:------:|:------:|
| -0.2143 | -0.3617 | -0.5439 | -0.3238 | -0.0271 |

> 正值 = 涨幅压倒跌幅，类似 RSI 但用了幅度而非天数。

### 信号解读

> SUMD = SUMP - SUMN，涨跌幅度的净值。与 CNTD 类似但看的是幅度而非天数。

| SUMD 值 | 含义 |
|:------:|------|
| > +0.4 | 涨幅远压倒跌幅，强势 |
| +0.1 ~ +0.4 | 涨幅略大于跌幅 |
| -0.1 ~ +0.1 | 涨跌幅度平衡 |
| -0.4 ~ -0.1 | 跌幅略大于涨幅 |
| < -0.4 | 跌幅远压倒涨幅，弱势 |

#### 天数因子 vs 幅度因子的区别

> CNTP/CNTN/CNTD 看**天数**（涨/跌了几天），SUMP/SUMN/SUMD 看**幅度**（涨/跌了多少钱）。两者可能给出不同信号：比如 5 天中涨了 3 天（CNTP=0.6）但涨幅很小、跌了 2 天（CNTN=0.4）但跌幅巨大，此时 SUMP 可能 < 0.5。这种分歧本身就是有用的信息。

### 3.24 VMA — 成交量均线（量能因子）

**直觉**：过去 N 天平均成交量相对于今日成交量。衡量今日是否异常放量/缩量。

```
VMA_N = Mean($volume, N) / ($volume + 1e-12)
```

| VMA5 | VMA10 | VMA20 | VMA30 | VMA60 |
|:----:|:-----:|:-----:|:-----:|:-----:|
| 1.0446 | 0.9604 | 0.9172 | 0.9035 | 1.1117 |

> < 1 = 今天放量（比均量大），> 1 = 今天缩量

### 信号解读

> VMA 衡量今日成交量相对于 N 日均量的比值。与 MA 因子的逻辑相同：分子是历史均量，分母是今日量。

| VMA 值 | 含义 |
|:------:|------|
| < 0.5 | 今日放量超过均量 2 倍，异常活跃 |
| 0.5 ~ 0.8 | 今日明显放量 |
| 0.8 ~ 1.2 | 今日量能正常（与均量接近） |
| 1.2 ~ 2.0 | 今日明显缩量 |
| > 2.0 | 今日极度缩量，市场冷清 |

> **注意**：VMA 只衡量量能大小，不区分涨跌方向。需配合价格因子判断是“放量上涨”还是“放量下跌”。

### 3.25 VSTD — 成交量波动（量能因子）

**直觉**：过去 N 天成交量的标准差，衡量量能的不稳定性。

```
VSTD_N = Std($volume, N) / ($volume + 1e-12)
```

| VSTD5 | VSTD10 | VSTD20 | VSTD30 | VSTD60 |
|:-----:|:------:|:------:|:------:|:------:|
| 0.3768 | 0.2760 | 0.2644 | 0.2980 | 0.4951 |

### 信号解读

> VSTD 衡量过去 N 天成交量的标准差，反映量能的不稳定性。

| VSTD 值 | 含义 |
|:------:|------|
| < 0.2 | 量能非常稳定，每天成交量差不多 |
| 0.2 ~ 0.4 | 量能正常波动 |
| 0.4 ~ 0.6 | 量能波动较大，有时放量有时缩量 |
| > 0.6 | 量能极不稳定，忽大忽小 |

> **与 VMA 配合**：VMA 看“今天量能大不大”，VSTD 看“量能稳不稳”。VMA 低（放量）+ VSTD 高（波动大）= 今天异常放量，值得关注。

### 3.26 WVMA — 量加权价格波动（波动因子）

**直觉**：日收益率的绝对值 × 成交量，再算标准差与均值之比。衡量"量价波动的一致性"。

```
WVMA_N = Std(|$close/Ref($close,1)-1| * $volume, N) / Mean(|$close/Ref($close,1)-1| * $volume, N)
```

| WVMA5 | WVMA10 | WVMA20 | WVMA30 | WVMA60 |
|:-----:|:------:|:------:|:------:|:------:|
| 1.1083 | 1.0660 | 0.8469 | 1.1238 | 2.1762 |

### 信号解读

> WVMA = 量加权价格波动的标准差/均值，衡量“量价共振的稳定性”。波动越大表示量价配合越不稳定。

| WVMA 值 | 含义 |
|:------:|------|
| < 0.5 | 量价波动非常稳定，市场有序 |
| 0.5 ~ 0.8 | 量价配合正常 |
| 0.8 ~ 1.2 | 量价波动较大，市场分歧明显 |
| > 1.2 | 量价极不稳定，可能是恐慌或异动 |

> **与 STD 的区别**：STD 只看价格波动，WVMA 将成交量纳入考量（收益率绝对值 × 成交量）。放量大涨和缩量微涨对 STD 的贡献相同，但对 WVMA 的贡献不同。

### 3.27 VSUMP — 放量幅度占比 / 量比 RSI（量能动量因子）

**直觉**：过去 N 天中成交量增加的天数，占成交量变化绝对值总和的比例。类似成交量的 RSI。

```
VSUMP_N = Sum(max($volume-Ref($volume,1), 0), N) / Sum(|$volume-Ref($volume,1)|, N)
```

| VSUMP5 | VSUMP10 | VSUMP20 | VSUMP30 | VSUMP60 |
|:------:|:-------:|:-------:|:-------:|:-------:|
| 0.5764 | 0.4909 | 0.5315 | 0.5072 | 0.5124 |

### 信号解读

> VSUMP 类似 SUMP 但针对成交量：衡量放量幅度占总变化的比例。可视为“成交量的 RSI”。

| VSUMP 值 | 含义 |
|:------:|------|
| > 0.7 | 放量幅度远大于缩量，资金持续流入 |
| 0.5 ~ 0.7 | 放量多于缩量，资金偏流入 |
| = 0.5 | 放缩量幅度相等，平衡 |
| 0.3 ~ 0.5 | 缩量多于放量，资金偏流出 |
| < 0.3 | 缩量幅度远大于放量，资金持续流出 |

### 3.28 VSUMN — 缩量幅度占比（量能动量因子）

```
VSUMN_N = 1 - VSUMP_N
```

| VSUMN5 | VSUMN10 | VSUMN20 | VSUMN30 | VSUMN60 |
|:------:|:-------:|:-------:|:-------:|:-------:|
| 0.4236 | 0.5091 | 0.4685 | 0.4928 | 0.4876 |

### 信号解读

> VSUMN = 1 - VSUMP，与 VSUMP 互补。

| VSUMN 值 | 含义 |
|:------:|------|
| > 0.7 | 缩量幅度远大于放量，资金持续流出 |
| 0.5 ~ 0.7 | 缩量多于放量，资金偏流出 |
| = 0.5 | 放缩量幅度相等 |
| 0.3 ~ 0.5 | 放量多于缩量，资金偏流入 |
| < 0.3 | 放量幅度远大于缩量，资金持续流入 |

### 3.29 VSUMD — 量能净差（量能动量因子）

```
VSUMD_N = VSUMP_N - VSUMN_N
```

| VSUMD5 | VSUMD10 | VSUMD20 | VSUMD30 | VSUMD60 |
|:------:|:-------:|:-------:|:-------:|:-------:|
| 0.1528 | -0.0181 | 0.0631 | 0.0143 | 0.0247 |

### 信号解读

> VSUMD = VSUMP - VSUMN，放缩量的净值。

| VSUMD 值 | 含义 |
|:------:|------|
| > +0.4 | 放量远压倒缩量，资金强势流入 |
| +0.1 ~ +0.4 | 放量略多于缩量 |
| -0.1 ~ +0.1 | 放缩量平衡 |
| -0.4 ~ -0.1 | 缩量略多于放量 |
| < -0.4 | 缩量远压倒放量，资金强势流出 |

#### VSUMP/VSUMN/VSUMD 与 SUMP/SUMN/SUMD 的类比

> 价格侧的 SUMP/SUMN/SUMD 衡量涨跌的幅度关系，量能侧的 VSUMP/VSUMN/VSUMD 衡量放缩量的幅度关系。两者配合可判断“资金流向与价格方向是否一致”：

| 组合 | 含义 |
|------|------|
| SUMP>0.5 + VSUMP>0.5 | 价格上涨且资金流入，趋势健康 |
| SUMP>0.5 + VSUMP<0.5 | 价格上涨但资金流出，可能无量拉升 |
| SUMP<0.5 + VSUMP>0.5 | 价格下跌但资金流入，可能是洗盘 |
| SUMP<0.5 + VSUMP<0.5 | 价格下跌且资金流出，趋势恶化 |
```

## 五、Alpha360 因子（360个）

Alpha360 是纯原始价格序列，不做任何衍生计算：

```
60 天 × 6 个字段 = 360 个因子

字段: CLOSE, OPEN, HIGH, LOW, VWAP, VOLUME
每天的值除以当日收盘价（价格）或当日成交量（成交量）做归一化。

CLOSE59, CLOSE58, ..., CLOSE1, CLOSE0  (CLOSE0 = 当日收盘/当日收盘 = 1)
OPEN59,  OPEN58,  ..., OPEN1,  OPEN0
HIGH59,  HIGH58,  ..., HIGH1,  HIGH0
LOW59,   LOW58,   ..., LOW1,   LOW0
VWAP59,  VWAP58,  ..., VWAP1,  VWAP0
VOLUME59,VOLUME58,...,VOLUME1,VOLUME0
```

> Alpha360 直接把过去 60 天的原始价格喂给深度学习模型（LSTM/Transformer），让模型自己学时序模式，不需要人工设计因子。

## 六、Qlib 表达式算子速查

编写自定义因子时常用的算子：

| 算子 | 语法 | 说明 |
|------|------|------|
| 引用原始字段 | `$close`, `$open`, `$high`, `$low`, `$volume`, `$vwap` | 基础字段 |
| 前 N 期引用 | `Ref($close, N)` | N 日前的收盘价 |
| N 期均值 | `Mean($close, N)` | 过去 N 日收盘均价 |
| N 期标准差 | `Std($close, N)` | 过去 N 日标准差 |
| N 期求和 | `Sum($close, N)` | 过去 N 日累加 |
| N 期最大值 | `Max($high, N)` | N 日内最高价 |
| N 期最小值 | `Min($low, N)` | N 日内最低价 |
| 线性回归斜率 | `Slope($close, N)` | β 系数 |
| 线性回归 R² | `Rsquare($close, N)` | 拟合度 |
| 线性回归残差 | `Resi($close, N)` | 最新残差 |
| 分位数 | `Quantile($close, N, p)` | p 分位数 (0~1) |
| 截面排名 | `Rank($close, N)` | 在 N 日中的百分位 |
| 最大值位置 | `IdxMax($high, N)` | 距最高点的天数 |
| 最小值位置 | `IdxMin($low, N)` | 距最低点的天数 |
| 相关性 | `Corr($close, $volume, N)` | 两序列 N 日相关性 |
| 对数 | `Log($volume + 1)` | 自然对数 |
| 条件 | `Greater(a, b)`, `Less(a, b)` | 返回 a 或 b 中较大/较小值 |
| 绝对值 | `Abs(x)` | 绝对值 |
| 四则运算 | `+`, `-`, `*`, `/` | 加减乘除 |
| 比较运算 | `>`, `<`, `==`, `!=` | 返回布尔值（True=1, False=0） |

## 七、标签（Label）定义

模型要预测的不是股票价格，而是 **T+1 的相对收益率**（明天的涨跌幅）：

```
LABEL0 = Ref($close, -2) / Ref($close, -1) - 1
```

**理解 Qlib 中 Ref 的索引方向**（这是最容易混淆的地方）：

| 表达式 | 含义 | 示例 |
|--------|------|------|
| `$close` | **今日** 收盘价 | 7.703750 |
| `Ref($close, 1)` | **1 日前**（昨天）的收盘价 | 7.782040（09-18） |
| `Ref($close, -1)` | **1 日后**（明天）的收盘价（未来） | — |
| `Ref($close, -2)` | **2 日后**（后天）的收盘价（未来） | — |

> **注意**：`Ref` 中 **正数** = 向过去回溯，**负数** = 向未来看。这是 Qlib 表达式引擎的约定，与 pandas 的 `shift()` 相反。

**公式分解**：

```
LABEL0 = Ref($close, -2) / Ref($close, -1) - 1
       = 后天收盘价(T+2) / 明天收盘价(T+1) - 1
       = T+1 到 T+2 的 1 天收益率
```

> **为什么叫"T+1 收益率"？** 因为站在 T 日看，标签预测的是"从明天(T+1)到后天(T+2)"的收益，即你明天买入后第一天的收益。

**以浦发银行 2020-09-21 为例**：

| 字段 | 日期 | 值 |
|------|------|-----|
| 明天收盘 (`Ref($close, -1)`) | 2020-09-22 | 7.703750 |
| 后天收盘 (`Ref($close, -2)`) | 2020-09-23 | 7.594144 |

```
LABEL0 = 7.594144 / 7.703750 - 1
       = 0.985771 - 1
       = -0.014229
```

> 标签含义：从 09-22 收盘到 09-23 收盘，浦发银行跌了约 **1.4%**。

**为什么用 `Ref($close, -2)/Ref($close, -1) - 1` 而不是直接用明天价格？**

- 除以 T+1 收盘价做**归一化**，消除不同股价量级的影响（茅台 1800 元 vs 银行股 5 元）
- `-1` 使其成为 **收益率**，便于截面比较
- 正值 = 明天上涨，负值 = 明天下跌

### 7.1 标签的真实含义：不是"今天到明天"，而是"明天到后天"

公式中的分子和分母**都是收盘价**，中间只隔 1 天：

```
LABEL0 = Ref($close, -2) / Ref($close, -1) - 1
       = (T+2 收盘价)   / (T+1 收盘价)   - 1
       = 从 T+1 收盘到 T+2 收盘的 1 天收益率
```

**不是 2 天**：分子和分母分别是后天和明天的收盘价，间隔只有 1 天。

**以连续 5 天为例**（站在 09-21 看）：

| 日期 | 收盘价 | 说明 |
|------|:------:|------|
| 09-21（T） | 10.00 | 今天，算因子、出信号 |
| 09-22（T+1） | 10.20 | 明天，分母 = `Ref($close,-1)` |
| 09-23（T+2） | 10.50 | 后天，分子 = `Ref($close,-2)` |
| 09-24（T+3） | 10.30 | — |
| 09-25（T+4） | 10.80 | — |

```
站在 09-21 的标签 = 10.50 / 10.20 - 1 = +2.94%
                    └── 09-22收盘 → 09-23收盘，1天收益 ──┘

站在 09-22 的标签 = 10.30 / 10.50 - 1 = -1.90%
                    └── 09-23收盘 → 09-24收盘，1天收益 ──┘

站在 09-23 的标签 = 10.80 / 10.30 - 1 = +4.85%
                    └── 09-24收盘 → 09-25收盘，1天收益 ──┘
```

> 每一行都只预测 **1 天**的收益，只是站在不同日期时日历日期不同。

### 7.2 为什么是"明天到后天"而不是"今天到明天"？

因为实盘中**今天收盘后你来不及买入**：

```
T 日收盘后 ──→ 算因子、模型出信号 ──→ T+1 开盘才能买入
                                          ↓
                                    持有到 T+1 收盘
                                          ↓
                                    T+2 收盘卖出（或继续持有）

你真正能赚到的第一段收益：
  买入价 ≈ T+1 收盘价（用收盘价近似代替开盘价，更稳定）
  卖出价 ≈ T+2 收盘价

所以标签 = close(T+2) / close(T+1) - 1
        = Ref($close,-2) / Ref($close,-1) - 1
```

> **注意**：标签用的是收盘价而非开盘价，所以 T+1 当天开盘到收盘的涨跌（日内收益）**没有包含在内**。这是一个近似设计，因为收盘价更稳定，且模型学的是截面排名而非绝对收益。

### 7.3 为什么用收益率而不是绝对价格？

- 除以当日收盘价做**归一化**，消除不同股价量级的影响（茅台 1800 元 vs 银行股 5 元）
- `-1` 使其成为**收益率**，便于截面比较
- 正值 = 明天上涨，负值 = 明天下跌

### 7.4 标签预处理：从原始收益到模型输入

原始标签算出来后，还需要经过 3 步预处理才会被模型看到。

#### 为什么必须预处理？

不同交易日的市场环境差异巨大：

```
牛市日：所有股票标签都是 +2% ~ +8%
熊市日：所有股票标签都是 -6% ~ -1%
震荡日：所有股票标签在 -2% ~ +2% 之间
```

如果直接用原始值训练，模型会被"今天大盘涨跌"主导，而不是学到"哪只股票更强"。

#### Step 1：截面 Z-Score 标准化（核心步骤）

**同一天的所有股票**，减均值、除标准差：

假设某天 5 只股票的原始标签：

| 股票 | 原始标签 | 标准化计算 | 标准化标签 | 含义 |
|------|:------:|------|:--------:|------|
| 茅台 | +2.0% | (2.0-(-0.4))/3.1 | **+0.77** | 比市场平均好 0.77 个标准差 |
| 平安 | +3.0% | (3.0-(-0.4))/3.1 | **+1.10** | 全市场最强 |
| 银行A | 0.0% | (0.0-(-0.4))/3.1 | **+0.13** | 接近平均 |
| 银行B | -2.0% | (-2.0-(-0.4))/3.1 | **-0.52** | 比平均差 |
| 小盘股 | -5.0% | (-5.0-(-0.4))/3.1 | **-1.48** | 全市场最弱 |

> 均值 = -0.4%，标准差 ≈ 3.1%

**第二天（大牛市，所有股票都涨了）**：

| 股票 | 原始标签 | 标准化后 |
|------|:------:|:------:|
| 茅台 | +5.0% | +0.80 |
| 平安 | +6.0% | +1.20 |
| 银行A | +2.0% | -0.40 |
| 银行B | +1.5% | -0.60 |
| 小盘股 | +3.0% | +0.00 |

虽然原始标签全是正的（都涨了），但标准化后：
- 平安 +1.20 → **跑赢大盘**（应该买入）
- 银行B -0.60 → **虽然涨了，但跑输大盘**（不该买）

> **截面标准化的本质**：消除"今天大盘涨跌"的影响，只保留"谁比谁强"的相对排名。

#### Step 2：Clipping（截断极端值）

```
截断到 [-3, 3]：
  +0.77 → 不变（在范围内）
  +5.20 → +3.00（防止极端值主导模型训练）
```

#### Step 3：Fillna（填充缺失）

某只股票停牌 → 标签 NaN → 填 0（视为"表现平平"）

### 7.5 标签全链路总览

从原始数据到实盘交易的完整流程：

```
┌─────────────────────────────────────────────────────────────┐
│                    训练阶段（同一天全市场）                     │
│                                                             │
│  股票A: 158个因子 ──→ 特征预处理 ──→ X_A                     │
│  股票B: 158个因子 ──→ 特征预处理 ──→ X_B                     │

│                                                             │
│  股票A: 原始标签 +3%  ─┐                                    │

| 阶段 | 标签的角色 | 说明 |
|------|-----------|------|
| **训练** | 作为 y（目标变量） | 模型学习"什么因子组合 → 跑赢市场" |

| **实盘** | 间接使用 | 按预测排名选 Top N 股票，T+1 开盘买入 |

## 八、数据预处理流水线

从因子公式计算出原始值后，Qlib 会对 **特征** 和 **标签** 分别做预处理。模型实际看到的是 **预处理后的值**，而非本手册中列出的公式原始值。

### 8.1 特征预处理（Alpha158）

按顺序执行以下三步：

#### 第一步：Robust Z-Score 标准化（逐股、逐因子）

```
标准化值 = (原始值 - 中位数) / (MAD × 1.4826)
```

其中 `MAD` = Median Absolute Deviation（绝对离差中位数）。

**为什么不用普通 Z-Score（均值+标准差）？**

金融数据常有极端值（涨跌停、异常波动），均值和标准差对这些极端值非常敏感。中位数和 MAD 更 **鲁棒（robust）**。

> 常数 1.4826 的作用：对于正态分布，`MAD × 1.4826 ≈ σ`，使得 Robust Z-Score 的值域近似于普通 Z-Score。

#### 第二步：Shrink（软截断/缩尾）

不是简单地把超过 ±3 的值截掉，而是 **压缩**（保留排序信息）：

```
if x > 3:   x = 3 + (x - 3) / (x_max - 3) × 0.5
if x < -3:  x = -3 + (x + 3) / (x_min + 3) × 0.5
```

| 原始值 | 普通截断（硬） | Shrink（软截断） |
|:------:|:------------:|:---------------:|
| 2.5 | 2.5 | 2.5 |
| 4.0 | 3.0 | 3.5 左右（被压缩但不丢失相对大小） |
| 10.0 | 3.0 | 3.8 左右 |

> **优势**：保留了极端值之间的排序关系，不像硬截断把 4.0 和 10.0 都变成 3.0。

#### 第三步：缺失值填充

所有 NaN 填为 0。

---

### 8.2 标签预处理

比特征处理简单：

| 步骤 | 操作 | 说明 |
|:--:|------|------|
| 1 | **截面 Z-Score 标准化** | 每个交易日，所有股票的标签减均值除标准差 |
| 2 | **Clipping** | 截断到 [-3, 3] |
| 3 | **Fillna** | 缺失值填 0 |

> **截面标准化** 意味着：在同一天，所有股票标签的均值变为 0，标准差变为 1。这使得不同交易日之间的标签具有可比性。

---

### 8.3 预处理对模型的影响

```
你的因子公式 → 原始值（手册中的数值）
                     │
        ┌────────────┴────────────┐
        ▼                         ▼
   特征预处理                   标签预处理
   RobustZScore + Shrink       ZScore + Clip
        │                         │
        ▼                         ▼
   预处理后的 X                 预处理后的 y
        │                         │
        └────────┬────────────────┘
                 ▼
            Model.fit(X, y)
```

| 要点 | 说明 |
|------|------|
| **手册因子值是原始值** | 模型实际训练用的是预处理后的值 |
| **预处理是逐股计算的** | 每只股票独立标准化，不是全市场混合 |
| **不同因子的数值不可直接比** | 原始值量纲不同（KMID ≈ ±0.01，RANK ≈ 0~1），标准化后量纲统一 |
| **Shrink 防止过拟合** | 极端值被压缩，模型不会被个别异常点牵着走 |

## 十、因子间关系速记

### 完全线性依赖（互为冗余）

| 关系 | 说明 |
|------|------|
| `SUMN = 1 - SUMP` | 跌幅占比 = 1 - 涨幅占比 |
| `VSUMN = 1 - VSUMP` | 缩量占比 = 1 - 放量占比 |
| `CNTP + CNTN ≤ 1` | 涨天数 + 跌天数 ≤ 总天数（可能有平盘） |

### 可互相推导

| 关系 | 说明 |
|------|------|
| `SUMD = SUMP - SUMN = 2×SUMP - 1` | 涨跌幅净差由 SUMP 唯一确定 |
| `CNTD = CNTP - CNTN` | 涨跌天数净差 |
| `VSUMD = VSUMP - VSUMN = 2×VSUMP - 1` | 量能净差由 VSUMP 唯一确定 |

### 常见因子组合解读

| 组合 | 解读 |
|------|------|
| ROC5 < 1 + RANK5 高 | 短期上涨且处于高位（可能超买） |
| RSQR60 高 + BETA60 > 0 | 长期稳健上升趋势 |
| CORR20 > 0 + CORD20 > 0 | 量价配合良好（价涨量增） |
| VMA20 < 1 + CORD20 > 0 | 放量上涨（健康信号） |
| STD20 高 + RSQR20 低 | 高波动、无趋势（震荡市） |
| MAX60 ≈ 1 + RANK60 高 | 接近 60 日新高（突破信号） |
| IMIN60 ≈ 0 + RSQR60 高 | 刚创 60 日新低且有趋势（可能继续跌） |

## 十一、因子按市场特征分类

每个因子都在衡量市场的**某一个方面**，没有因子是"万能"的。理解因子背后的金融逻辑，才能知道什么时候该用、什么时候该信。以下按 7 大市场特征将 158 个因子归类。

### 11.1 动量因子（Momentum）— 「趋势会延续吗？」

> 核心假设：强者恒强，弱者恒弱。过去涨的股票未来继续涨。

> 适合模型：趋势跟踪策略。在趋势市中表现好，震荡市中容易反复打脸。

| 因子前缀 | 因子数 | 衡量内容 | 关键解读 |
|----------|:------:|----------|----------|
| **ROC** | 5 | N 日前价格相对于今天的涨跌 | ROC<1=近期上涨，ROC>1=近期下跌 |
| **BETA** | 5 | 线性趋势的每日涨跌速度 | BETA>0=上升通道，BETA<0=下降通道 |
| **CNTP** | 5 | 上涨天数占比 | >0.5=偏多，<0.5=偏空 |
| **CNTN** | 5 | 下跌天数占比 | 与 CNTP 互补 |
| **CNTD** | 5 | 涨跌天数净差 | 正值=做多力量强 |
| **SUMP** | 5 | 涨幅幅度占比（类 RSI） | >0.5=上涨比下跌有力 |
| **SUMN** | 5 | 跌幅幅度占比 | <0.5=下跌比上涨弱 |
| **SUMD** | 5 | 涨幅-跌幅净值 | 正=多头占优 |
| **VSUMP** | 5 | 放量幅度占比 | >0.5=量能偏多 |
| **VSUMN** | 5 | 缩量幅度占比 | 与 VSUMP 互补 |
| **VSUMD** | 5 | 量能净差 | 正=放量时期多于缩量 |

> **小计：55 个因子**（11 种算子 × 5 窗口）

**需要注意的陷阱**：
- 动量因子在**趋势转折点**往往给出错误信号（涨太多了反而跌）
- 需要结合反转因子（RANK、RSV、RESI）来判断是否超买/超卖

---

### 11.2 趋势因子（Trend）— 「价格在走直线吗？」

> 核心假设：趋势明确的股票比震荡的股票更容易预测，也更有交易价值。

> 适合模型：趋势跟随 + 趋势过滤。趋势强的股票加大仓位，震荡的股票避开。

| 因子前缀 | 因子数 | 衡量内容 | 关键解读 |
|----------|:------:|----------|----------|
| **MA** | 5 | 收盘价相对 N 日均线的位置 | MA<1=收盘在均线上方（多头排列） |
| **RSQR** | 5 | 价格走直线的程度（R²） | 接近1=趋势流畅，接近0=震荡无序 |
| **IMAX** | 5 | 距上次 N 日新高的天数（Aroon上轨） | 接近0=刚创新高，接近1=长期未创新高 |
| **IMIN** | 5 | 距上次 N 日新低的天数（Aroon下轨） | 接近0=刚创新低，接近1=长期未创新低 |
| **IMXD** | 5 | 高点和低点的先后顺序 | 正=先涨后跌（见顶），负=先跌后涨（见底） |
| **MAX** | 5 | 当前价距 N 日最高点的距离 | 接近1=在最高点附近 |
| **MIN** | 5 | 当前价距 N 日最低点的距离 | 接近1=在最低点附近 |

> **小计：35 个因子**（7 种算子 × 5 窗口）

---

### 11.3 反转因子（Reversal/超买超卖）— 「该反弹了吗？」

> 核心假设：价格偏离均值太远会回归。涨多了要跌，跌多了要涨。

> 适合模型：均值回归策略。在震荡市中表现好，趋势市中容易被趋势淹没。

| 因子前缀 | 因子数 | 衡量内容 | 关键解读 |
|----------|:------:|----------|----------|
| **RANK** | 5 | 当前收盘价在 N 日内的百分位 | >0.8=超买区（可能回调），<0.2=超卖区（可能反弹） |
| **RSV** | 5 | 收盘价在 N 日高低区间的相对位置 | 0=收在最低（超卖），1=收在最高（超买），是 KDJ 的基础 |
| **RESI** | 5 | 当前价偏离趋势线的残差 | 正值=高于趋势线（可能超买），负值=低于趋势线（可能超卖） |

> **小计：15 个因子**（3 种算子 × 5 窗口）

**这三个因子的区别**：
- **RANK** 看的是"当前价在历史上的排位"（统计意义），不看价格形态
- **RSV** 看的是"当前价在 N 日箱体里的位置"（技术意义），考虑了最高最低形成的支撑阻力
- **RESI** 看的是"当前价偏离趋势线多少"（趋势意义），偏离过大可能回归

---

### 11.4 波动率因子（Volatility）— 「风险有多大？」

> 核心假设：高波动的股票风险更大，但机会也可能更多。低波动的股票更稳定。

| 因子前缀 | 因子数 | 衡量内容 | 关键解读 |
|----------|:------:|----------|----------|
| **STD** | 5 | 收盘价的标准差（除以当前价去量纲） | 值越大=波动越剧烈 |
| **KLEN** | 1 | 当日振幅 | 日内最高最低的波动幅度 |
| **WVMA** | 5 | 量加权的价格波动稳定性 | 值越大=量价波动越不稳定 |
| **VSTD** | 5 | 成交量波动 | 值越大=量能越不稳定 |
| **QTLU** | 5 | 80% 分位数相对当前价 | 衡量价格分布的上沿 |
| **QTLD** | 5 | 20% 分位数相对当前价 | 衡量价格分布的下沿 |

> **小计：26 个因子**（1 + 5×5）

**应用场景**：
- 选股时过滤掉 STD 过高的股票（控制回撤风险）
- 期权定价中波动率是核心输入
- QTLU 和 QTLD 的差值近似于 N 日波动区间

---

### 11.5 量价配合因子（Volume-Price）— 「钱在跟着价格走吗？」

> 核心假设："量为价先"。有量支撑的上涨比无量上涨更可靠。

| 因子前缀 | 因子数 | 衡量内容 | 关键解读 |
|----------|:------:|----------|----------|
| **CORR** | 5 | 收盘价与成交量的相关性 | 正=价涨量增（健康），负=价涨量缩（背离） |
| **CORD** | 5 | 日收益率与日量比的相关性 | 正=涨时放量跌时缩量（健康资金流） |

> **小计：10 个因子**（2 种算子 × 5 窗口）

**CORR 与 CORD 的区别**：
- **CORR** 用原始价格和原始成交量，衡量的是"价格水平和成交量水平"的关系
- **CORD** 用日收益率和日成交量变化率，衡量的是"涨跌方向和放缩量方向"的关系，更加精细

---

### 11.6 量能因子（Volume）— 「市场在关注这只股票吗？」

> 核心假设：成交量反映市场参与度和关注度。异常放量往往意味着有大事发生。

| 因子前缀 | 因子数 | 衡量内容 | 关键解读 |
|----------|:------:|----------|----------|
| **VMA** | 5 | 今日成交量相对于 N 日均量 | <1=今日放量（异常活跃），>1=今日缩量（冷清） |
| **VSTD** | 5 | N 日成交量波动 | 值大=量能忽大忽小（不稳定） |
| **VSUMP** | 5 | 放量幅度占总变化的比例 | 同动量中的量能方向 |
| **VSUMN** | 5 | 缩量幅度占总变化的比例 | 同上 |
| **VSUMD** | 5 | 量能净差 | 同上 |

> **小计：25 个因子**（5 种算子 × 5 窗口）

> 注：VSUMP/VSUMN/VSUMD 同时属于动量因子和量能因子，避免重复计数，这里归入动量。

---

### 11.7 形态因子（Pattern/K 线形态）— 「今天这根 K 线长什么样？」

> 核心假设：单根 K 线的形态包含当日多空博弈的痕迹，长下影线 = 有人护盘，长上影线 = 有人出货。

| 因子 | 因子数 | 衡量内容 | 关键解读 |
|------|:------:|----------|----------|
| **KMID** | 1 | 实体涨跌幅 | 正=阳线（收>开），负=阴线 |
| **KLEN** | 1 | 全日振幅 | 值大=多空博弈激烈 |
| **KMID2** | 1 | 实体占比 | 0=十字星（多空平衡），±1=光头光脚（一边倒） |
| **KUP** | 1 | 上影线长度 | 上影长=冲高回落（上方有抛压） |
| **KUP2** | 1 | 上影线占比 | 同上，消除振幅影响 |
| **KLOW** | 1 | 下影线长度 | 下影长=探底回升（下方有支撑） |
| **KLOW2** | 1 | 下影线占比 | 同上，消除振幅影响 |
| **KSFT** | 1 | 收盘在最高最低之间的位置 | 正=偏上方收盘（强势），负=偏下方收盘（弱势） |
| **KSFT2** | 1 | 收盘位置占比 | -1=收在最低，0=居中，+1=收在最高 |
| **OPEN0** | 1 | 开盘相对收盘 | >1=高开低走（弱势），<1=低开高走（强势） |
| **HIGH0** | 1 | 最高相对收盘 | 反映盘中冲高程度 |
| **LOW0** | 1 | 最低相对收盘 | 反映盘中下探程度 |
| **VWAP0** | 1 | 均价相对收盘 | >1=均价高于收盘（尾盘走弱），<1=均价低于收盘（尾盘走强） |

> **小计：13 个因子**
